Security Architecture

The Iron Curtain.
Not a metaphor.

Three non-negotiable architectural layers that make SchemaDock the only SQL assistant governance teams will approve on their first review.

Pillar 01

Read-Only Database Isolation

SchemaDock connects exclusively to a designated read-replica. Connection strings are vault-stored and never exposed. DML statements are syntactically blocked before reaching the query engine — not filtered; structurally impossible.

  • Dedicated replica credentials
  • DML/DDL structural block layer
  • Query rate limits & timeout caps
Pillar 02

In-Flight PII Redaction Engine

Column-level classification runs on schema metadata, not data content. Emails, SSNs, card numbers, and custom regex patterns are masked in query results before the response payload reaches Slack's API surface.

  • Schema-level PII labeling
  • Configurable custom redaction rules
  • Zero raw PII in logs or transit
Pillar 03

Local Infrastructure-First Proxy

The SchemaDock proxy agent runs inside your VPC. Raw query results never traverse the public internet. Only structured, redacted summary payloads are transmitted outbound — fully auditable via your existing SIEM stack.

  • VPC-resident agent, self-hosted
  • SIEM-compatible audit logs
  • mTLS encrypted outbound only
Compliance coverage: SOC 2 Type II GDPR Art. 25 HIPAA ISO 27001 CCPA
